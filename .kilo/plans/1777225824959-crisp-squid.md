# Plan: Shoot on tap in top third while right-area dragging

## Goal
When a finger is already dragging on the right side of the screen (camera control), a short tap with a second finger in the top third of the screen should fire a shot. The reload button retains priority over the shoot tap.

## Changes (single file: `src/TouchController.js`)

### 1. Add `shootTap` state (constructor)
Add a new state object to track the optional second-finger shoot tap:
```js
this.shootTap = {
    active: false,
    touchId: null,
    startX: 0,
    startY: 0,
    movedDistance: 0,
    cancelled: false
};
```

### 2. Update `handleTouchStart` — detect second finger tap
After the existing reload-button check, add logic for when `rightArea` is already active and a new touch lands on the right half in the top third of the screen (excluding the reload button):

- If `this.rightArea.active` is true AND `x >= halfX` AND `y < (window.innerHeight / 3)` AND NOT inside reload button → assign this touch as `shootTap` action, store start position.

This goes between the reload check and the left-stick check, or after the rightArea check as an additional branch.

### 3. Update `handleTouchMove` — cancel if finger drags too far
If the touch matches `shootTap.touchId`, track movement distance. If it exceeds `tapThreshold`, mark `shootTap.cancelled = true`.

### 4. Update `handleTouchEnd` — fire shot on clean tap
If the action is `shootTap` and it was NOT cancelled:
- Set `this.triggerShoot = true` and `this.triggerButton = true`

Then reset `shootTap` state.

### 5. Update `reset()` — clear `shootTap`
Reset `shootTap.active = false`, `shootTap.touchId = null`, etc.

## Priority order in handleTouchStart
1. Reload button check (existing, highest priority)
2. Left stick (existing)
3. Right area activation (existing, when not already active)
4. **NEW:** Shoot tap detection (when rightArea already active, top third, not reload button)
