# Plan: Fire Button on Left Joystick Release

## Goal
When the player lifts their finger from the left joystick after walking, a temporary "fire" button appears at that exact screen position for 3 seconds. If the next walk starts by touching this button instead of the regular left-half area, the player shoots immediately on movement start.

## Changes

### 1. `src/TouchController.js` — Add fire button state

**Add new properties to `constructor` (after line 64):**
```js
this.fireButton = {
    active: false,
    x: 0,
    y: 0,
    radius: 30,
    createdAt: 0,
    lifetime: 3000  // 3ms
};
```

**Modify `handleTouchEnd` (lines 250-264):**
After the left stick is released AND it was actual movement (not a tap), spawn the fire button:
```js
// Inside the leftStick touch-end block, before resetting state:
if (this.leftStick.movedDistance >= this.tapThreshold) {
    // This was actual movement, spawn fire button at release position
    this.fireButton.active = true;
    this.fireButton.x = this.leftStick.baseX;
    this.fireButton.y = this.leftStick.baseY;
    this.fireButton.createdAt = Date.now();
}
```

**Add `isInsideFireButton(x, y)` method** (similar to `isInsideReloadButton`):
```js
isInsideFireButton(x, y) {
    if (!this.fireButton.active) return false;
    const dx = x - this.fireButton.x;
    const dy = y - this.fireButton.y;
    return dx * dx + dy * dy <= this.fireButton.radius * this.fireButton.radius;
}
```

**Add `updateFireButton()` method** (called from Game each frame to check expiry):
```js
updateFireButton() {
    if (this.fireButton.active && (Date.now() - this.fireButton.createdAt > this.fireButton.lifetime)) {
        this.fireButton.active = false;
    }
}
```

**Add a flag to controller data for shoot-on-move:**
The `getControllerData()` returns data each frame. We need to signal "shoot on start of movement". We'll use a new approach: set a flag `shootOnMoveStart` that gets consumed once.

Actually, simpler approach: when a touch starts on the fire button, immediately fire the shot AND start the joystick. No need for a separate flag.

**Modify `handleTouchStart` (lines 133-148):**
Before the generic left-half joystick check, check if the touch lands on the fire button:
```js
// Check fire button BEFORE generic left-stick assignment
if (this.isInsideFireButton(x, y) && !this.leftStick.active) {
    this.triggerShoot = true;
    this.triggerButton = true;
    // Fall through to normal joystick setup below
    // (fire button deactivates on use)
    this.fireButton.active = false;
}
```

This way the fire button check happens after reload button check (line 125), Fires the shot via the existing trigger mechanism, then the touch falls through to the normal left-stick assignment on line 133.

**Modify `render()` method (line 335-348):**
Draw the fire button when active:
```js
// Кнопка огня
if (this.fireButton.active) {
    this.drawFireButton(ctx);
}
```

**Add `drawFireButton(ctx)` method:**
```js
drawFireButton(ctx) {
    const btn = this.fireButton;
    const elapsed = Date.now() - btn.createdAt;
    const remaining = 1 - (elapsed / btn.lifetime);
    const alpha = remaining; // Fade out over 3 seconds
    
    ctx.save();
    ctx.globalAlpha = alpha;
    
    // Outer ring
    ctx.beginPath();
    ctx.arc(btn.x, btn.y, btn.radius, 0, Math.PI * 2);
    ctx.fillStyle = 'rgba(255, 80, 50, 0.3)';
    ctx.fill();
    ctx.strokeStyle = 'rgba(255, 100, 50, 0.8)';
    ctx.lineWidth = 2;
    ctx.stroke();
    
    // Timer ring (shrinking arc)
    ctx.beginPath();
    ctx.arc(btn.x, btn.y, btn.radius + 4, -Math.PI / 2, -Math.PI / 2 + Math.PI * 2 * remaining);
    ctx.strokeStyle = 'rgba(255, 120, 50, 0.9)';
    ctx.lineWidth = 3;
    ctx.stroke();
    
    // Icon (fire symbol or bullet)
    ctx.fillStyle = 'rgba(255, 200, 100, 0.9)';
    ctx.font = 'bold 24px Arial';
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText('●', btn.x, btn.y + 1);
    
    ctx.restore();
}
```

**Modify `reset()` method:**
Add reset of fire button:
```js
this.fireButton.active = false;
```

### 2. `src/Game.js` — Update fire button timer

**In `update()` method (around line 954, after `updateTouchController()`):**
```js
this.touchController.updateFireButton();
```

This ensures the fire button expires after 3 seconds even if no touches occur.

| File | Changes |
|------|---------|
| `src/TouchController.js` | Add fire button state, hit detection, spawn on left-stick release, consume on touchstart, render with fade+timer ring, auto-expire |

## Flow
1. Player walks with left joystick → lifts finger → `handleTouchEnd` fires
2. If `movedDistance >= tapThreshold` (was real movement, not a tap): fire button spawns at `baseX, baseY`
3. Fire button renders with fade-out over 3 seconds, disappears when timer expires
4. Next `touchstart` on left side: if inside fire button → `triggerShoot = true` fires immediately, then normal joystick setup proceeds
5. Player moves AND shoots simultaneously from the start of the movement
